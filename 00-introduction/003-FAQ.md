## Geral

### O que é o LOCKDOWN?

LOCKDOWN é uma plataforma de governança para comunidades Discord. Não é apenas um bot, mas um ecossistema distribuído que fornece moderação em tempo real e aplicação de políticas em múltiplos servidores.

### Quais são os quatro pilares?

1. **AUTH** - Identidade e controle de acesso
2. **MODERATION** - Ordem e bem-estar da comunidade
3. **SECURITY** - Prevenção e defesa
4. **TICKETS** - Comunicação estruturada

### Qual a stack tecnológica?

- Backend: Express.js + TypeScript
- Bots: Discord.js
- Frontend: React + Vite
- Database: SQLite (dev) / PostgreSQL (prod)
- ORM: Drizzle
- Cache/Events: Redis
- Build: Turborepo + pnpm

## Instalação

### Como instalo o projeto?

```bash
git clone git@github.com:Mickael-Virtuoso/LOCKDOWN.git
cd LOCKDOWN
pnpm install
cp .env.example .env
pnpm dev
```

### Preciso de Docker?

Não é obrigatório, mas recomendado para PostgreSQL e Redis em desenvolvimento.

### Qual versão do Node.js?

Node.js 18.17.0 ou superior.

## Desenvolvimento

### Como crio um novo endpoint?

1. Crie o schema em `database/schemas/`
2. Crie o repository em `database/repositories/`
3. Crie o service em `database/services/`
4. Crie o controller em `api/controllers/`
5. Adicione a rota em `api/routes/`

Veja [BACKEND.md](./BACKEND.md) para detalhes.

### Como crio um novo comando de bot?

```typescript
@Command({ name: 'meucomando' })
export default class MeuComando extends BaseCommand {
  async execute(interaction) {
    // Implementação
  }
}
```

Veja [BOT.md](./BOT.md) para detalhes.

### Como rodo os testes?

```bash
pnpm test              # Todos os testes
pnpm test:unit         # Apenas unit tests
pnpm test:integration  # Apenas integration
pnpm test:e2e          # End-to-end
```

## Database

### SQLite ou PostgreSQL?

- **Desenvolvimento**: SQLite (mais simples)
- **Produção**: PostgreSQL (mais robusto)

### Como faço migrações?

```bash
pnpm db:migrate        # Aplicar migrações
pnpm db:studio         # Abrir Drizzle Studio
```

### Os dados são compartilhados entre servidores?

Sim, através do sistema de Shared Ban Lists e Redis Pub/Sub.

## Troubleshooting

### O bot não conecta

1. Verifique `DISCORD_BOT_TOKEN`
2. Verifique se o bot está no servidor
3. Verifique os intents no Discord Developer Portal

### Redis não conecta

1. Verifique se Redis está rodando
2. Verifique `REDIS_HOST` e `REDIS_PORT`
3. Verifique firewall

### Erro de TypeScript

```bash
pnpm type-check  # Verificar erros
pnpm clean       # Limpar cache
pnpm install     # Reinstalar dependências
```
